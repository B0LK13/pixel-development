Findings
--------
1. Duplicate clean-clone work: the §8 nested run re-executes the ENTIRE
   suite inside the clone (109s ≈ half the suite). This is by design: it is
   the hermetic proof that a fresh clone is self-verifying. Removing or
   thinning it would remove the proof, so it is KEPT (charter: do not remove
   the clean-clone proof). PIXEL_TESTS_NO_CLONE=1 remains the documented
   fast path for iteration (1m50s).
2. No unnecessary sleeps: the only deliberate waits are the §23 signal-test
   stub (sleep 3, bounded by a 10s poll) and genuine 1s timeout(1) kills,
   which are the behaviour under test.
3. No repeated lint passes: bash -n and shellcheck each run once per file.
   Gap closed this session: §1/§2 previously linted only the 4 root scripts +
   harness; they now cover all 8 tracked shell scripts (scripts/*.sh were
   previously only linted via ci-local.sh). Cost: ~1-2s; value: real.
4. Repeated fixture setup: each autodev e2e test builds its own git
   workspace (~0.4s × ~15). A shared template+cp could save ~5-8s but would
   touch plumbing used by 19 tests for a ~4% suite win — rejected on
   risk/reward (correctness takes priority).
5. External command overhead: gpg keygen (2×) is the only >1s tool startup;
   already minimal (ed25519, no passphrase).

Before/after
------------
  session-4 tip: 146 tests, ~2m45s
  session-5:     199 tests,  3m31s  (+53 tests, all real coverage; no
                 optimization removed anything; the only structural cost is
                 the chartered nested clone proof)
Decision: no optimizations applied. Correctness and coverage took priority;
the budget is documented and the fast iteration path
(PIXEL_TESTS_NO_CLONE=1) is preserved.
